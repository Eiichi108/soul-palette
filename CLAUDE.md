## ■ Claudeへの共通指示

1. **言語**: TypeScript必須。`any`型は使わない
2. **スタイル**: Tailwind CSSのみ。ただし動的なRGB値（個体値カラー等）はインラインstyle許容
3. **状態管理**: React hooks（useState / useEffect / useContext）のみ
4. **Supabase**: `src/lib/supabase.ts` のクライアントを使う
5. **エラー処理**: try/catch必須。エラーはUIにも表示
6. **コメント**: WHYが非自明な箇所のみ日本語で記載
7. **ファイル分割**: 1ファイル200行超えたら分割を検討
8. **スマホ対応**: モバイルファーストで実装
9. **不明点**: 不明な点は思い込みで実装せず、必ずユーザーに質問してから実装する。
10. **CLAUDE.md更新**: 新機能追加・仕様変更・ファイル構成変更を実装完了後に、該当箇所（実装済み機能サマリー・ファイル構成・ゲーム仕様など）をCLAUDE.mdに反映する。

## ■ ゲーム仕様

### 職業・基礎ステータス（Lv1）

| 職業 | HP | ATK | DEF | SPD |
|------|-----|-----|-----|-----|
| 戦士（job_id=1） | 350 | 250 | 300 | 100 |
| 魔法使い（job_id=2） | 240 | 400 | 200 | 120 |
| 武闘家（job_id=3） | 280 | 280 | 280 | 160 |
| 弓使い（job_id=4） | 270 | 320 | 240 | 170 |

- **成長式**: `Math.floor(base + (base * 4 / 49) * (level - 1))` → Lv50で約5倍
- **個体値**: iv_hp / iv_atk / iv_def（0〜255）をステータスに直接加算。SPDのIVなし
- **個体値カラー**: `rgb(iv_atk, iv_hp, iv_def)`（R=ATK, G=HP, B=DEF）
- **レベル上限**: デフォルトLv50、合成で+5ずつ最大Lv100

### ダメージ計算式

```
ダメージ = (22 × 威力 × ATK ÷ DEF) ÷ 50 + 2
回復量   = (威力 × ATK) ÷ 500
クリティカル = ×1.5（基本確率5%、スキルで変動）
```

### バトルルール

- SPD順に行動、自分のターンにパッシブ以外の全スキルを発動
- 最大30ターン → 引き分け
- 一方のパーティ全滅で勝敗決定
- 報酬: 勝利EXP+100/Gold+50、引き分けEXP+30/Gold+10、敗北EXP+10
- UI: 4カラムグリッド（敵上・味方下）、DQスタイル1件ログ、スキップ機能

### 状態異常

| 種別 | 効果 | 重複 |
|------|------|------|
| 毒 | 毎ターンHP減少 | 同種不可 |
| 麻痺 | 毎ターン行動不能判定 | 同種不可 |
| スタン | 1ターン行動不能 | 同種不可 |
| SPDデバフ | SPD低下 | 複数可 |
| シールド | ダメージ吸収（HP扱いではない） | 同種不可 |
| 封印 | パッシブスキル無効化 | 同種不可 |
